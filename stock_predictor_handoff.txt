FRED MACRO DASHBOARD
--------------------
/api/macro-dashboard fetches 11 FRED series in parallel via ThreadPoolExecutor(max_workers=6).

FRED API call format:
  https://api.stlouisfed.org/fred/series/observations
    ?series_id=CPIAUCSL&frequency=m&aggregation_method=eop&limit=26&sort_order=desc
    &api_key=<FRED_API_KEY>&file_type=json

Uses urllib.request (stdlib only) — no external library needed.

11 indicators:
  CPI        CPIAUCSL   %    yoy    good=down
  CORE_CPI   CPILFESL   %    yoy    good=down
  PCE        PCEPI      %    yoy    good=down
  UNRATE     UNRATE     %    raw    good=down
  PAYROLLS   PAYEMS     K    mom    good=up
  10Y        DGS10      %    raw    good=neutral
  2Y         DGS2       %    raw    good=neutral
  M2         M2SL       %    yoy    good=neutral
  FED_FUNDS  FEDFUNDS   %    raw    good=neutral
  CFNAI      CFNAI      idx  raw    good=up
  SENTIMENT  UMCSENT    idx  raw    good=up

CHAOS FILTER (meme/high-vol stock exclusion)
---------------------------------------------
Applied in both _train_single() (scanner) and get_prediction() (single lookup).
Rule: avg ATR over last 30 trading days > 7% → skip
  - Scanner: silently returns None
  - Single lookup: returns signal="EXCLUDED" with excluded_reason field
Rationale: 10-day swing model is structurally wrong fit for meme stocks (HOOD, GME, etc.)
Prior threshold was 5% — raised to 7% on Apr 27 because post-Black Swan ATR spiked
across legitimate stocks too.


MACRO PREDICTED PAGE (5th tab)
--------------------------------
Endpoint: GET /api/macro-score  (cache: 2h)
Returns a bull_score 0–100 derived from 14 macro indicators:
  VIX, YIELD_CURVE, CPI, CORE_CPI, PCE, UNRATE, PAYROLLS,
  FED_FUNDS, M2, CFNAI, SENTIMENT, BTC, OIL, GOLD

Scoring: each indicator scored -100 to +100 via tier lookup (_bull_score_item).
Weights sum to 100. Final: (weighted_avg + 100) / 2 → [0, 100].
Regime labels: ≥75 Strong Bull, ≥60 Moderate Bull, ≥45 Neutral,
               ≥30 Cautious Bear, else Strong Bear.

Known bug: BZ=F (Brent crude) sometimes returns stale/wrong price via yfinance.
           Not yet fixed.

Frontend: MacroPredictedView + MacroScoreCard components at end of App.jsx.
Gauge: gradient bar red→green with needle at score position.
